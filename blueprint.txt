Enterprise Workflow Orchestration & Decision Platform

Primary Goal

Build a configurable workflow orchestration platform capable of:

Dynamic Workflow Definition
Dynamic Bucket Creation
Dynamic Context Resolution
Dynamic Rule Evaluation
Dynamic Routing
Transaction Replay
Explainability

without requiring code changes for normal business modifications.

Simple Rule Engine
Drools UI Wrapper

Instructions To Google Antigravity
Must Follow
Workflow definitions are source of truth.
Runtime instances are immutable snapshots.
Context resolution must be lazy.
Rules must never directly invoke integrations.
Execution journal must drive debugger.
Published versions must be immutable.
Buckets are first-class entities.
Runtime execution model must be DAG based.
Every decision must be explainable.
Every execution must be replayable.
Most Important Rule

If a design decision must be made, always prefer:

Maintainability
Explainability
Replayability
Configurability

over:

Short-Term Simplicity
